## [v0.2.0] - 2026-07-16

### Highlights

- Render providers for all three vendors: generate a talking-head MP4 from narration audio via HeyGen, Tavus, or bitHuman
- HeyGen and bitHuman render providers implement `render.AudioUploader` for uploading local narration files
- Symmetric registry API: `GetLiveProvider` / `GetRenderProvider`

### Added

- Render registry API: `RegisterRenderProvider`, `GetRenderProvider`, `ListRenderProviders`, `HasRenderProvider`, `GetRenderProviderPriority` ([`9a3c4d4`](https://github.com/plexusone/omniavatar/commit/9a3c4d4))
- Render surface for all three vendors — audio-driven talking-head generation via HeyGen, Tavus, and bitHuman. The render adapters live in the provider SDK repos (`heygen-go/omniavatar`, `tavus-go/omniavatar`, `bithuman-go/omniavatar`), depending only on `omniavatar-core`; the batteries `providers/{heygen,tavus,bithuman}` packages register them alongside the live adapters ([`9a3c4d4`](https://github.com/plexusone/omniavatar/commit/9a3c4d4))
- HeyGen and bitHuman render adapters implement `render.AudioUploader`; the HeyGen adapter implements `render.AvatarLister` (v2 avatars) ([`9a3c4d4`](https://github.com/plexusone/omniavatar/commit/9a3c4d4))
- All providers register both live and render surfaces via `providers/all` ([`9a3c4d4`](https://github.com/plexusone/omniavatar/commit/9a3c4d4))
- `render-basic` example generating a talking-head video from narration audio via any registered render provider ([`5461d66`](https://github.com/plexusone/omniavatar/commit/5461d66))
- Unit tests for the live/render registry surfaces ([`8e6e9aa`](https://github.com/plexusone/omniavatar/commit/8e6e9aa))

## [v0.1.0] - 2026-07-12

### Highlights

- Initial release with HeyGen, Tavus, and bitHuman providers
- Priority-based provider registry with auto-registration via init()
- LiveKit token generation for avatar participants

### Added

- `RegisterAvatarProvider()` for registering avatar provider factories
- `GetAvatarProvider()` for retrieving registered providers by name
- `ListAvatarProviders()` for discovering available providers
- `HasAvatarProvider()` for checking provider registration
- `GenerateAvatarToken()` for creating LiveKit tokens with avatar metadata
- `LiveKitStartOptions` struct for platform-specific session initialization
- `PriorityThin` (0) and `PriorityThick` (10) registration priority constants
- `providers/heygen`: HeyGen LiveAvatar provider with LITE mode support
- `providers/tavus`: Tavus Conversational Video provider with PAL support
- `providers/bithuman`: bitHuman Real-time Avatars provider
- `providers/all`: Convenience blank import for all providers

### Documentation

- README.md with architecture, quick start, and provider details
- Package documentation in doc.go with usage examples
- docs/releases/v0.1.0.md release notes
